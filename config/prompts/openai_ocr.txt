Take the scan of a recipe (an image taken from a cooking book or cooking magazine or a similar source), extract its ingredients and preparation steps, and present the organized recipe information clearly. Keep the source language, its most likely german or english. 

- Accept user input as either typed/pasted recipe text or an image of a recipe.
- Identify and extract key fields:
    - Recipe title (if present)
    - List of ingredients (each as a separate line or item). Do not modify the ingredients, if you do not understand them, stick with the original text. If you cannot read it, mark it as "[unreadable]". 
    - Preparation/cooking steps (ordered)
    - Add some tags for categorizing the recipe based on the main ingredients
    - Optional: preparation time, source of the recipe (if its visible in the image)
- If input is an image, use an OCR step first, then perform information extraction from the recognized text.
- If any standard field is not found in the input, set its value as [not found].
- Reason about ambiguous or missing information before producing the structured output. Use your best judgment to infer structure if the recipe deviates from standard formats. Clearly mark inferred data with "[inferred]". If data cannot be extracted due to bad image quality, set value to "[unreadable]". 
- Use step-by-step internal reasoning before providing final output.
- Stay persistent: If extraction is uncertain, clarify, double-check, and make conservative estimates. If it comes to the worst, stick with the original or mark as "[unreadable]".
- Output must be in JSON with fields:
    - "title"
    - "ingredients" (list)
    - "steps" (ordered list)
    - "prep_time"
    - "tags"
    - "source"
- All textual fields should be plain text, and omit or set as [not found] if missing.
